## 🔎 프로젝트 소개
여러분의 결혼 이야기를 우리 **Marry View**에서 허심탄회하게 표출해보세요!  
**MarryView**는 **스튜디오 - 드레스 - 메이크업**에 대한 정보를 실제 경험자들의 후기를 통해  
결혼을 준비하고 있는 **예비 부부**들에게 **실제 선배 부부**들이 제공해주는 정보로 더욱 신뢰할 수 있고,  
업체 상품을 연계해주는 하이브리드 웨딩 플랫폼입니다.

- 현재 결혼한 부부들의 스-드-메 결혼 후기를 제공
- 영수증 인증 기반 **관리자 승인제**를 통한 리뷰 업로드로 신뢰도 확보
- 허위/광고성 리뷰 원천 차단
- 마음에 드는 후기 확인 후 원하는 상품 예약 연계

## 🧠 주요 기능

- 다중 권한 기반 회원 시스템: 일반 사용자, 업체, 관리자별 Role(역할) 부여 및 차별화된 기능 제공

- 통합 마이페이지: 게시글, 댓글, 좋아요, 신고/문의 내역 등 사용자 활동 데이터 요약 및 관리

- 카카오맵 API 연동: 위치 기반 서비스(찾아오시는 길) 및 업체 정보 제공

- 예약 및 문의 프로세스: 상품별 예약 서비스, 실시간 문의 내역 조회 및 상태 관리

- 리치 에디터 커뮤니티: 사진 첨부 기능이 포함된 게시글 작성 및 실시간 댓글 상호작용

- 비즈니스 모델(Paid Content): 리뷰 유·무료 구분(블러 처리), 영수증 인증 기반의 관리자 승인 시스템

- 파일 관리 및 보안: Cloudinary를 활용한 미디어 서버 관리 및 민감 서류(영수증) 접근 권한 제어

- 결제 시스템: 결제 API 연동을 통한 이용권(Pass) 구매 및 잔여 횟수 관리

## 💪🏼 역할 분담

 ### 임효진 (팀장)
- **작업 페이지:**

   - 업체 페이지 전체 (상품관리,예약관리,문의내역,리뷰내역)
   - 상품 목록, 업체 목록 전체 (상품예약,문의)

- **구현 주요 기능:**
  
  - 스마트 예약·결제 및 Q&A 프로세스 연동: '상품 선택 → 예약 저장 → 내역 확인 → 결제 API 연동'으로 이어지는 주문 흐름 및 문의-답변 양방향 데이터 트래킹 구현
  - 리뷰 권한별 조건부 렌더링 및 페이징: 유료/무료 리뷰 권한별 화면 노출 차별화(이미지 제공 및 5줄 요약 제한) 및 Front-end 중심의 리뷰 페이징 처리
  - 대용량 상품 정보 처리 및 다중 태그 필터링: 멀티파트 업로드 기반 이미지 서버 경로 매핑 및 MyBatis foreach를 활용해 사용자가 선택한 여러 개의 복합 태그를 AND 조건으로 정교하게 교집합 검색하는 다중 필터링 기능 구현
  - 사용자 접근 시점 기반 상태 자동 동기화: 사용자 접근 시점(예약/업체 조회)의 날짜·결제 상태를 비교하여 예약 상태(DONE/CANCEL)를 실시간 업데이트하는 로직 구축
  - 컴포넌트 기반 아키텍처 최적화: 대형 페이지(3,000줄)를 단일 책임 원칙(SRP)에 맞춰 구조화된 독립 컴포넌트(50~200줄)로 분리하여 앱 로딩 및 유지보수 성능 개선
  - 결제 상황별 권한 기반 접근 제어: 결제 상황(대기, 확정, 취소, 만료)에 따른 세밀한 서비스 접근 권한 구분 및 CS 처리 상태(대기/완료) 시각화
 
 ### 안혜진
- **작업 페이지:**

   - 마이페이지 전체, 대문(INTRO), 메인 홈, 헤더 백엔드, 로그인, 회원가입, AI 챗봇

- **구현 주요 기능:**

  - 마이페이지 전체 페이징 구현 — XML LIMIT/OFFSET 처리, Controller에서 페이징 계산 후 Vue로 데이터 전달
  - 열람 내역 조회 — 유료/무료 탭 분리, pass_usage_log 기반 유료 리뷰 열람 기록 조회
  - 예약/결제/쿠폰 내역 조회 — 다중 테이블 JOIN으로 사용자별 내역 출력
  - 마이페이지 내 글/리뷰/댓글/좋아요 목록 — 탭 전환 방식으로 한 페이지에서 통합 관리, 체크박스 선택 삭제 구현
  - 회원가입/로그인 — 일반 회원과 업체 회원 분리 처리
  - AI 챗봇 — LangChain + Gemini 기반 웨딩 상담 챗봇 연동

 ### 최제현
- **작업 페이지:**

   - 관리자페이지 전체, 회사 연혁, 패스구매,결제, 패스 조회, 제휴등록 결제, 결제 완료 페이지

- **구현 주요 기능:**

  - 패스 구매 - 외부 결제 API(PortOne)를 활용한 결제 및 결제 완료 후 결제 위변조 방지를 위한 서버 검증 로직. Vue.js를 활용한 동적 UI 관리 및 쿠폰 선택에 따른 실시간 할인 금액 계산
  - 내 패스 조회 -  결제, 쿠폰 내역을 통합관리하고 필터링, 페이징, 모달 기능을 구현함으로써 사용자 중심의 결제 이력 관리
  - 회원/업체, 상세페이지 - @change와 AJAX를 활용한 비동기 동적 필터링으로 회원 목록을 효율적으로 검색, 필터링, 정렬하고 상세정보를 확인하며 계정상태를 제어할수 있는 기능
  - 게시판/리뷰 페이지 - 비동기 동적 필터링을 통한 게시글, 리뷰 목록을 검색, 필터링, 정렬. 페이지 이동 없는 편리한 조회. soft delete방식을 사용 하는 컨텐츠 검수 시스템
  - 결제 - 상태변수와 v-if를 이용해 만든 탭을 이용한 결제 내역 통합관리. 외부 api를 활용한 환불처리.
  - 상품 페이지 - 상태변수와 v-if를 이용한 탭 UI. @click과 AJAX를 사용하여 서버에 기록된 상품상태 변경 및 조회.
  - 통계 - AJAX를 활용한 데이터 로딩 및 반복문을 이용한 차트용 데이터로 재가공. 차트라이브러리(ApexCharts)를 활용한 월별 통계 시각화. 차트 동적 생성 및
    재렌더링을 통한 차트 중복 및 메모리 누수 방지.
    
 ### 이태화
- **작업 페이지:**

   - 커뮤니티, 리뷰 ,신고, 문의

- **구현 주요 기능:**
  
  - 커뮤니티와 리뷰 게시글에서 에디터로 사진 첨부, 리뷰 게시글에서는 에디터로 첨부한 첫 번째 사진의 경로를 추출해서 경로로 만들어 리뷰 목록에 출력
  - 리뷰 게시글 작성 시 영수증 파일은 서버에 저장하는 API 사용 후 Spring Boot에서 File첨부 업로드 로직을 작성해 관리자에게만 출력하도록 구현
  - 리뷰 게시글의 대한 열람권 차감 로직 구현 -> 무료 리뷰는 전부 보여주고 유료 리뷰는 열람권 소유 여부 유효성 검사를 통해 유료 리뷰 조회 가능
